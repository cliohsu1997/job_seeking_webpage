## Understanding the Reports

### Summary Report (`diagnostics_summary_latest.txt`)
Quick overview with:
- Total issues count
- Issues by category (validation, normalization, parsing)
- Most common errors
- Issues by source

### Full Report (`diagnostics_full_latest.json`)
Complete diagnostic data including:
- Summary statistics
- All issues with details
- Root cause analysis
- Category statistics

### Category Reports
Detailed issues for specific categories:
- **Validation Issues**: Schema validation errors, missing required fields, invalid data formats
- **Normalization Issues**: URL resolution problems, date parsing errors, text cleaning issues
- **Parsing Issues**: File read errors, HTML/XML parsing failures

## How to Use

1. **Quick Check**: Read `diagnostics_summary_latest.txt` for a high-level overview
2. **Detailed Analysis**: Open `diagnostics_full_latest.json` for complete data
3. **Specific Issues**: Check category-specific JSON files for targeted problems
4. **Historical Comparison**: Compare timestamped reports to track improvements

## Report Generation

Reports are automatically generated when running the processing pipeline:
```bash
poetry run python scripts/processor/pipeline.py
```

## File Management

**Important**: Only the latest diagnostic reports are kept in this directory.

- **Latest reports**: Files with `_latest` suffix (e.g., `diagnostics_summary_latest.txt`) always point to the most recent run
- **Timestamped reports**: Only the most recent timestamped files are kept (format: `diagnostics_*_YYYYMMDD_HHMMSS.*`)
- **Old reports**: Automatically cleaned up - old timestamped files are deleted when new reports are generated
- **This ensures**: The directory doesn't accumulate old diagnostic reports over time

See `read_it.md` for the complete diagnostic report management policy.
